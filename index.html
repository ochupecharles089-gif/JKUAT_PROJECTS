Paragraphs & Whitespace Collapse

The <p> element represents a paragraph of text. Browsers automatically add margin before and after each paragraph, creating visual separation without any extra markup.

This is a second paragraph. Notice the gap above — that's default browser styling. Multiple   spaces in HTML source are collapsed into a single space when rendered, as the notes explain.

Paragraphs cannot contain block-level elements like another <p> or a <div>, only inline content such as links, emphasis, or images.

Preformatted Text & Code Blocks

The <pre> element preserves every space, tab, and newline exactly as written in the source. Ideal for code snippets and ASCII art. 

function greet(name) {
    // Preserved indentation!
    console.log("Hello, " + name + "!");
}

greet("World");

Note: angle brackets inside <pre> must be escaped — &lt; for < and &gt; for >.
